Projects

Omega OS

This started as a thought experiment: what if you could build an operating system that was not just a collection of programs but an intelligence-driven environment? Omega OS is my attempt at designing a large-scale AI-powered system concept. It includes persistence layers, memory architecture, provider routing, safety protocols, automation pipelines, and deep AI integrations. It is not a product yet. It is an idea that I keep refining, a vision of what software could look like when it is designed with intelligence at its core.

Highlights: SQLite Persistence · Memory Architecture · Provider Routing · AI Infrastructure · Automated Systems

OpenKwen

A lightweight, open-source gateway built for developers who want to connect to any OpenAI-compatible provider without the usual setup overhead. I built OpenKwen because I was tired of rewriting the same integration code every time I wanted to test a new provider. It handles authentication, request routing, error handling, and response parsing out of the box. You point it at an endpoint, and it just works.

OpenGateway

This project came from a simple frustration: managing multiple API keys across dozens of AI providers is a mess. OpenGateway lets you use a single API key to access tons of different providers underneath. It acts as a unified layer between your application and the AI ecosystem. Rate limiting, fallbacks, provider switching, usage tracking, all handled in one place. I use this for almost everything I build now.

Kwen

The core engine behind a lot of my experiments. Kwen is a modular AI orchestration tool that I designed to handle complex multi-provider workflows. It started as a personal utility and slowly grew into something more structured. It handles prompt pipelines, context management, provider switching, and response post-processing. Not polished enough to call a product, but it works exactly the way I need it to.

Kwen App

A frontend interface built on top of the Kwen engine. I wanted a clean, minimal UI where I could interact with multiple AI models side by side, compare outputs, tweak parameters, and save conversations. It is nothing fancy visually, but it does exactly what I built it for. I use it daily for testing prompts and experimenting with different model behaviours.

Feio Library

A small, focused Python library that I wrote to handle the repetitive parts of working with AI models. Things like token counting, retry logic, response validation, and structured output parsing. I got tired of copying the same utility functions across different projects, so I bundled them into something I could import and forget about. It is not published anywhere officially, but it powers most of my backend work.

Feio Scriptures

A separate project, almost nothing like the others. I have been collecting and organizing philosophical and religious texts from different traditions, cleaning them up, and making them easier to read and reference. It started as a personal research thing, but I am slowly turning it into a standalone site. No AI, no gateways, no infrastructure. Just old words that still mean something.

RootBugs Labs

This is where messy ideas turn into something real. RootBugs Labs is my personal experimental space. No pressure to ship, no deadlines, no expectations. Just building for the sake of learning. Some ideas work. Most do not. But every experiment teaches me something new.
